# План ручной проверки

## Цель
Проверить бизнес-логику до подключения реальных сервисов.

## Проверить вручную
1. Создание нового заказа через API
2. Повторный заказ тем же клиентом в пределах 96 часов
3. Корректную классификацию в 600 / 900 / 1500+
4. Создание двух концепций главной страницы
5. Формирование approve / reject ссылок
6. Перевод заказа в approved
7. Финализацию проекта после approved
8. Формирование Divi 5 export и brief file
9. Работа короткого опросника без перегруза
10. Передача 1-3 референс-сайтов
11. Мягкая подача цены при дорогом референсе
12. Наличие budget-friendly альтернативы в reasoning

## Что не подключаем сейчас
- реальный WhatsApp
- реальный Telegram
- реальный email
- реальный OpenAI
- реальную production БД
- live URL crawler

## После ручной проверки подключаем
1. новую БД
2. новый GitHub repo
3. email provider
4. OpenAI provider
5. WhatsApp provider
6. Telegram bot
7. сервис анализа URL-референсов

## Дополнительная проверка многоязычности
1. Запустить intake на English, Deutsch, Francais, Italiano и Espanol.
2. Убедиться, что структура ответов в базе одинаковая независимо от языка интерфейса.
3. Убедиться, что при повторном визите в течение 96 часов по умолчанию предлагается ранее выбранный язык клиента.

## Новый ручной шаг approve/reject
Для локальной проверки без email-ссылок можно использовать endpoint:
- `POST /api/orders/{order_id}/decision/manual?action=approve`
- `POST /api/orders/{order_id}/decision/manual?action=reject`

После approve можно вызывать:
- `POST /api/orders/{order_id}/finalize?selected_concept_label=A`
- или `...selected_concept_label=B`

## Проверка референсов
- клиент вставляет 1-3 ссылки на понравившиеся сайты;
- ссылки сохраняются в заказе;
- комментарий клиента о том, что именно понравилось, сохраняется;
- GET /api/orders/{order_id} возвращает reference_sites;
- pricing_reasoning учитывает сигналы дорогой визуальной подачи, эффектов и popups;
- если сайт-референс выглядит как более дорогой уровень, reasoning объясняет это мягко и предлагает бюджетный вариант.


## Дополнительные ручные тесты по опроснику
1. Клиент вставляет только 2 референс-сайта — заказ создаётся.
2. Клиент не вставляет ссылки, а даёт только `desired_site_description` — заказ создаётся.
3. Клиент просит дорогой сайт с popup и анимациями только текстом — pricing повышается мягко, без жёсткого отпугивания.
4. Клиент выбирает бюджетный путь — система оставляет starter, если нет сигналов сложности.
