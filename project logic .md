I have a website hosted on Hostinger.
This website is dedicated to web development.
Our main goal is to quickly and beautifully create a demo page.
The website has a button (preview your new or future webpage in 60 seconds) for filling out fields. The first field is the website theme or your own website to be created. The second field accepts an email address, WhatsApp, Telegram, or Facebook Messenger. The user is redirected to the main website, either through an ad or a newsletter, and clicks a button prompting them to fill out two fields. The first is to specify their existing website or the theme for the new website. After submitting this information, the page is immediately generated online. The page should be "Wow!!! I want this!" The user can generate the page twice. After the third click, a button or online message appears prompting them to go to the main website and place an order. If the user doesn't fill out at least one field, the generate button won't appear. The generated page should be as secure as possible and difficult to change after it's uploaded. The generated page lasts 10 minutes and is then deleted. The generated page itself is stored in my database for 96 hours without protection so that I can retrieve it from the database for further processing if the user returns.
If a client provides their own website for generation, we only create the homepage. We leave the contact information, links, videos, and payment forms intact. Everything remains original. We simply rework the text, images, and the website itself to meet the requirements. Wow, I'd love to have that!
The security system should track data entered into fields. If fields are duplicated and the user has previously visited that field, they should be prompted to leave it and place their order. There should be one exception (klon97048@gmail.com) so that if the user returns after 96 hours, I can generate a new page for them after they pay for the order.

The generated demo page should contain a warning that it can only be generated twice, as well as a button with a link to the main website for placing an order. If the user enters their email address and then a subject or website address, the page is generated immediately. However, if the user enters a WhatsApp, Telegram, or Facebook Messenger number, a message is generated for them.
After clicking on it, the demo page is also generated. This is to prevent my WhatsApp, Telegram, or Facebook Messenger accounts from being blocked. The customer must send me a message first.
If the customer completes only one generation and leaves, they receive the message: "Try another generation."

If the customer completes two generations and then leaves or goes to another page after being redirected to the main page, they receive the message: "You have not completed your order on our website."

A WhatsApp bot and an Open AI bot will be installed on the main website.
The messages I send must be different each time so that WhatsApp, Telegram, and Facebook Messenger can verify that the user is a real person. The message can be generated in different languages: English, French, Italian, Spanish, German, Danish, Norwegian, Finnish, and Swedish. Depending on the region from which the user is visiting the site, the message is generated in the language of that region. If there are no such countries, the message is generated in English.
To create the project, we use OpenAI, GitHub, Railway, Supabase, Resend, Playwright, FastAPI, Vercel, Cloudflare Turnstile, Supabase Queues, PostHog, Sentry, and Hostinger.
